Visualization Pattern

Each node in the cluster is a single row. Every hardware resource on that node — GPUs, disks, NICs, IB interfaces, etc. — is represented by a small square. The square acts as a fill gauge: a colored region fills it from the bottom up, proportional to the metric's percentage value. An empty square means 0%; a fully filled square means 100%.

The fill color encodes severity. For most metrics (disk usage, RAM, temperature, network saturation), higher is worse, so the color grades from green (low/healthy) through yellow (moderate) to red (critical). GPU utilization is the exception — higher is better, so a full green square means the GPU is working hard and a nearly-empty red square means it's underutilized.

A half-filled yellow square immediately tells you "this resource is at ~50% and starting to warm up" without needing to read a number. The pattern lets you scan an entire cluster in a glance: a row of tall green fills = healthy node, a row of tall red fills = something needs attention. Hover any square for exact numbers. For metrics that are not naturally 0–100% (like temperature), a practical upper bound is chosen (e.g. 90°C for H100 thermal throttle) and the fill represents the fraction of that bound. The tooltip always shows the actual value / max so the denominator is explicit.

Tabs above each column let you switch which metric the squares display. For example, the GPU column can show utilization, VRAM usage, temperature, power draw, or job allocation. The square positions stay fixed (one per device, left to right by index) so you can flip between tabs and compare the same device across metrics.

Column Reference

Node
Hostname, IP address, and composite health status. The left accent bar indicates overall health: green = all subsystems healthy, yellow = one or more warnings, red = critical issue requiring attention.
GPUs
One square per GPU on the node (ordered by device index). Each square is a fill gauge showing the selected metric as a percentage.
  • Util — SM compute utilization (DCGM_FI_DEV_GPU_UTIL). Higher is better: 0% = idle (gray), >65% = green, 25–65% = yellow, <25% = red (underutilized).
  • VRAM — VRAM usage as % of total (e.g. 44.8 / 80 GB). Shown as a capacity fill gauge.
  • Tmp — GPU die temperature as a fraction of 90°C (thermal throttle ceiling for H100). Tooltip shows actual / max. <50°C green, 50–70°C yellow, >70°C red.
  • Pwr — Power draw as % of TDP (e.g. 320 / 700 W).
  • Job — Job allocation. Colored squares with a letter ID indicate which job owns each GPU. Dashed empty squares are unallocated.
Disks
One square per NVMe drive. The fill level represents space used — an empty square means mostly free (healthy), a full square means nearly out of space (critical). Hover for used/total in TB.
  • Space — Disk capacity consumed. <40% green, 40–75% yellow, >75% red. Hover shows used/total in TB.
  • IOPS — I/O operations per second as % of device maximum. High IOPS + high queue depth = I/O pressure.
  • Tput — Sequential read/write throughput as % of rated max (GB/s).
Network
One square per Ethernet NIC. Fill level = % of link bandwidth consumed.
  • BW — Aggregate TX+RX throughput as % of link speed (e.g. 85 / 100 Gb/s).
  • Drop — Packet drops / errors. Color is based on absolute count thresholds (>500 = red).
RDMA / IB
One square per InfiniBand HCA / RDMA interface. Critical for GPU-to-GPU communication in distributed training.
  • BW — Bandwidth utilization as % of link rate (e.g. 150 / 200 Gb/s HDR).
  • Cng — Fabric congestion measured by ECN (Explicit Congestion Notification) marks per second. High ECN = the switch fabric is pushing back.
  • Rty — Transport retries per second. A key signal of noisy-neighbor interference or link-level errors. >1000/s is critical.
PCIe
One square per node representing the aggregate PCIe root complex.
  • Tput — Total PCIe throughput as % of theoretical max (e.g. 28.4 / 63 GB/s for Gen5 x16). Hover shows per-device breakdown (GPU, NIC, NVMe).
  • Sat — PCIe saturation — how close the bus is to becoming a bottleneck. Factors in queuing and contention beyond raw throughput.
CPU / RAM
Two squares: first is CPU, second is RAM.
  • CPU — Overall CPU utilization across all cores + a RAM capacity gauge.
  • Mem — Physical memory usage as % of total (e.g. 1420 GB / 2 TB).
  • Swp — Swap usage + PSI (Pressure Stall Information). Any swap usage on a GPU node is a red flag. PSI >10% indicates memory pressure causing stalls.
